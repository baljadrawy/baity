## 🔴 يحتاج قرارات (لا يمكن إكمالها بدون مدخلات خارجية)

### 1. Web Push — تفعيل
**الحالة:** Backend كامل، UI toggle جاهز، ينقص VAPID keys فقط.

**الإجراء:**
```bash
docker run --rm node:22-alpine npx -y web-push generate-vapid-keys --json
```
ثم أضف للـ `.env.production`:
```
VAPID_PUBLIC_KEY=...
VAPID_PRIVATE_KEY=...
VAPID_SUBJECT=mailto:gjafagifts@gmail.com
```

**الفائدة:** إشعارات فورية على المتصفح حتى لو الموقع مغلق (تكميلية لـ Telegram).

---

### 2. Sentry — تفعيل
**الحالة:** الكود جاهز، DSN فارغ في `.env.production`.

**الإجراء:**
1. سجّل في [sentry.io](https://sentry.io) (مجاني للمشاريع الصغيرة)
2. أنشئ project بنوع Next.js
3. أضف `SENTRY_DSN` في `.env.production`

**الفائدة:** مراقبة الأخطاء في production — تعرف لو حد من المختبرين واجه bug بدون ما يخبرك.

---

### 3. BullMQ + Redis — Queue infrastructure
**الحالة:** الكود الحالي يعتمد على crontab بدائي. BullMQ يعطي retries + concurrency + delayed jobs.

**التكلفة:**
- Redis container جديد (~80MB image، ~50-150MB RAM)
- إعادة كتابة 5 cron jobs لتستخدم Workers
- ~2 ساعات عمل

**يحتاج قرار:** الـ Pi 4 عنده 8GB RAM. هل لديك مساحة؟

**التوصية:** ليس ضرورياً للـ MVP بـ 5 عوائل. يصير ضرورياً عند 50+.

---

### 4. OCR (Tesseract.js) — قراءة نصوص الفواتير
**الحالة:** غير مُنفَّذ. حقل `ocrText` في schema موجود لكن فارغ.

**التكلفة:**
- مكتبة 30MB+ — لكن server-side فقط، لا تأثير على bundle المتصفح
- ~3-10 ثوانٍ لكل صورة على Pi 4
- ~1.5 ساعة عمل

**الفائدة:**
- استخراج المبلغ + التاريخ من فاتورة مصوّرة تلقائياً
- بحث بالنص في الأرشيف

**يحتاج قرار:** هل تريد المعالجة async (background) أم sync (المستخدم ينتظر)؟ async أفضل UX.

---

### 5. HTTPS + Domain العام
**الحالة:** الوصول حالياً عبر Tailscale فقط. للإطلاق العام لازم:
- شراء domain (مثل `baity.app`)
- إعداد Caddy للـ TLS
- Port forwarding في الراوتر

**التكلفة:** ~50 ر.س/سنة domain + ساعة إعداد.

**يحتاج قرار:** متى تطلق علناً؟

---

### 6. النسخ الاحتياطي المشفّر
**الحالة:** نسخة يومية على نفس الـ Pi، بدون تشفير.

**الإجراء:** إضافة `age` encryption + رفع لـ S3/Backblaze.

**يحتاج قرار:** اشتراك تخزين سحابي (Backblaze B2 ~$0.005/GB/شهر — أقل من ريال شهرياً).

---

### 7. Income tracking (لو غيّرت رأيك)
**الحالة:** سابقاً اعترضت أن التطبيق ما يعرف الدخل. لو غيّرت رأيك:
- جدول `Income` (مصدر، مبلغ، تاريخ، متكرر؟)
- صفحة لإدخالها
- Monthly Flow chart يصير دخل + مصروف حقيقيان

**التوصية:** ابدأ بدون. لو طلبه المختبرون، أضفه.

## 🟢 جاهز للتنفيذ بدون قرار (متى تشاء)

### تحسينات قيمة (ساعة لكل ميزة)

#### A. رفع صور الفواتير
**ما هو:** `/api/v1/upload` موجود، لكن BillForm لا يستخدمه.
**الإضافة:** حقل صورة الفاتورة في الـ form + معاينة + ربط بـ `Bill.attachmentUrl`.
**أولوية:** **عالية** — ميزة شائعة، مطلب طبيعي للمختبرين.

#### B. Onboarding tour تفاعلي
**ما هو:** overlay ينطلق أول مرة يدخل المستخدم، يشير لكل قسم بـ tooltips متسلسلة.
**أولوية:** متوسطة — الـ help page تغطي الفجوة لكن ليست proactive.

#### C. Reports page (`/reports`)
**ما هو:** رسوم سنوية للفواتير + تقرير اقتصاد البيت + تصدير PDF.
**أولوية:** متوسطة — يعتمد على feedback المختبرين.

#### D. Calendar view (`/calendar`)
**ما هو:** كل المهام/الفواتير/الصيانة في تقويم موحد (هجري + ميلادي).
**مصدر الفكرة:** EXTRACTED_FEATURES — مذكور كـ "إلهام Homechart" + "إضافة سعودية" (مناسبات).
**أولوية:** متوسطة — يفيد لو كثير من البيانات.

#### E. Reminders مخصصة
**ما هو:** model `Reminder` + UI لإضافة تذكير عام (مثل "صيانة المكيف"، "تجديد الإقامة"، "موعد الطبيب").
**أولوية:** **عالية** — كثير من احتياجات البيت لا تندرج تحت "فاتورة" أو "مهمة دورية".

#### F. Meal Planning
**ما هو:** schema `Meal` + auto shopping list من المقادير.
**مصدر الفكرة:** EXTRACTED_FEATURES (Homechart) — Phase 3 بالخطة.
**أولوية:** منخفضة — متخصصة جداً.

#### N. Child Login مستقل
**ما هو:** صفحة `/child/login` بـ PIN-only (الـ API جاهز، فقط UI).
**أولوية:** **عالية** — حالياً الطفل يحتاج جلسة الوالد. Child Login مستقل = استقلالية كاملة.

#### O. Hijri Events — تنبيهات قبل المناسبات
**ما هو:** الآن يعرض المناسبات في widget. الإضافة: تنبيه Telegram "رمضان بعد 7 أيام".
**أولوية:** منخفضة — جميل لكن غير ضروري.

### تكاملات (تحتاج API keys / تجارية)

#### P. Email parsing بـ Claude API
**ما هو:** استلام فواتير عبر إيميل، استخراج البيانات بـ Claude API.
